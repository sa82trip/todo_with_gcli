# Gemini Project: React Todo App

## 1. 개요
- **목표**: 할 일(Todo)을 등록, 삭제, 조회할 수 있는 간단한 웹 애플리케이션을 제작합니다.
- **기술 스택**: React를 사용하며, 별도의 데이터베이스 없이 상태는 애플리케이션 메모리에서 관리합니다.

## 2. 개발 규칙
### 2.1. 언어
- **코드**: 변수명, 함수명, 주석 등 모든 코드는 **영어**로 작성합니다.
- **커밋 메시지**: **영어**로 작성합니다. (기억된 규칙)

### 2.2. 커밋 메시지 컨벤션
- 모든 커밋 메시지는 [Conventional Commits](https://www.conventionalcommits.org/) 명세를 따릅니다.
- 형식: `<type>(<scope>): <subject>`
- **주요 타입:**
  - `feat`: 새로운 기능 추가
  - `fix`: 버그 수정
  - `docs`: 문서 수정
  - `style`: 코드 포맷팅, 세미콜론 누락 등 (코드 변경은 없음)
  - `refactor`: 코드 리팩토링
  - `test`: 테스트 코드 추가/수정
  - `chore`: 빌드 관련 파일 수정, 패키지 매니저 설정 등

### 2.3. 브랜치 전략
- `main`: 최종 결과물이 반영되는 메인 브랜치입니다.
- `feature/{기능이름}`: 기능 개발을 위한 브랜치입니다. (예: `feature/add-todo-item`)
- `fix/{수정내용}`: 버그 수정을 위한 브랜치입니다.

### 2.4. 코딩 스타일
- **포맷팅**: Prettier를 사용하여 코드 스타일을 통일합니다.
- **린팅**: ESLint를 사용하여 잠재적인 오류나 안티패턴을 방지합니다.

### 2.5. 디렉토리 구조
- `src/components`: 공통으로 사용되는 리액트 컴포넌트
- `src/pages` 또는 `src/views`: 페이지 단위의 컴포넌트
- `src/hooks`: 커스텀 훅
- `src/styles`: 전역 스타일 또는 스타일 관련 파일

### 2.6. 상태 관리
- React Hooks (`useState`, `useReducer`, `useContext`)를 기본으로 사용합니다.
